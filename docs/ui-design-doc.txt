# TOEFL 阅读错因诊断系统 - 前端页面设计文档

**版本：** v1.1  
**更新日期：** 2025年1月

---

## 一、页面架构总览

系统共包含 3 个核心页面，采用单页应用（SPA）架构：

| 页面名称 | 路由 | 核心功能 |
|---------|------|---------|
| QuestionPage | `/question/:id` | 文章阅读、答题、结果展示 |
| ReflectionPage | `/reflection/:answerId` | 6步复盘流程 |
| DiagnosisPage | `/diagnosis/:answerId` | 诊断结果与能力提升建议 |

---

## 三、页面详细设计

### 3.1 QuestionPage（题目页）

#### 布局结构
采用**左右分栏布局**，比例为 50:50。

**左侧区域 - 文章展示区：**
- 顶部显示文章标题
- 主体为文章内容区，内容较长时支持独立滚动
- 背景色与右侧区分，便于阅读

**右侧区域 - 答题区：**
- 顶部显示题目题干
- 中部为 4 个选项，垂直排列，每个选项为可点击的卡片样式
- 底部为操作按钮区

#### 交互状态

**状态 A - 答题前：**
- 4 个选项均为默认样式（白色背景、灰色边框）
- 用户点击选项后，该选项高亮显示（蓝色边框）
- 底部显示「提交答案」按钮（蓝色主按钮）

**状态 B - 答题后（回答错误）：**
- 用户选择的错误选项变为红色样式：红色背景、红色边框、显示 ✗ 图标、右下角标注「你的选择」
- 正确选项变为绿色样式：绿色背景、绿色边框、显示 ✓ 图标、右下角标注「正确答案」
- 其他未选择的选项保持灰色，降低视觉权重
- 底部按钮变为「开始复盘」（蓝色主按钮）

**状态 C - 答题后（回答正确）：**
- 用户选择的选项变为绿色样式：绿色背景、绿色边框、显示 ✓ 图标、显示「正确！」提示
- 底部按钮变为「下一题」（蓝色主按钮）

### 3.2 ReflectionPage（复盘流程页）

#### 布局结构
采用**垂直布局**，从上到下分为 4 个区域。

**区域 1 - 可折叠的题目信息区：**
- 默认状态为折叠，显示一行提示文字「▶ 查看题目和文章」
- 点击后展开，采用左右分栏显示：
  - 左侧：文章标题和内容摘要
  - 右侧：题目题干、用户的错误选择（红色标记）、正确答案（绿色标记）
- 再次点击可收起，便于用户专注于复盘步骤

**区域 2 - 进度指示条：**
- 水平排列 6 个步骤节点，用连线串联
- 已完成的步骤显示为实心圆点（蓝色）
- 当前步骤显示为实心圆点（蓝色）并带有高亮效果
- 未完成的步骤显示为空心圆点（灰色）
- 每个节点下方显示步骤简称：定位词识别、答案句定位、句子理解、选项理解、自我诊断、补充说明

**区域 3 - 步骤内容卡片区（核心区域）：**
- **采用卡片式设计**，每个步骤的内容呈现在一张独立的卡片中
- 卡片居中显示，有明显的阴影和圆角，视觉上突出
- 卡片内容包括：步骤标题（如「Step 2/6 答案句定位」）、引导问题、选项列表或输入框
- **预留划走动画接口**：卡片支持左右滑动手势，向右划表示确认并进入下一步，向左划表示返回上一步

**区域 4 - 底部导航按钮：**
- 左侧「上一步」按钮（灰色次按钮），第一步时禁用或隐藏
- 右侧「下一步」按钮（蓝色主按钮），最后一步时变为「提交复盘」

#### 各步骤卡片内容

**Step 1 - 定位词识别：**
- 引导问题：「请选择你认为的定位词：」
- 交互方式：单选，4 个候选词/短语
- 卡片内显示完整题干供参考

**Step 2 - 答案句定位：**
- 引导问题：「请选择你认为的答案句：」
- 交互方式：单选，3-4 个候选句子
- 每个选项为完整句子，可能需要多行显示

**Step 3 - 答案句理解：**
- 引导问题：「请选择最接近你理解的选项：」
- 交互方式：单选 + 条件输入
- 提供 3 个预设的理解模板选项
- 第 4 个选项为「以上都不对，我的理解是：」，选中后在卡片内展开文本输入框

**Step 4 - 选项理解（分为 Part A 和 Part B）：**
- Part A - 错误选项理解：
  - 卡片顶部显示用户选择的错误选项
  - 引导问题：「请选择最接近你当时理解的选项：」
  - 提供 3-4 个预设模板，描述常见的错误思维路径
  - 最后一个选项支持自由输入
- Part B - 正确选项理解：
  - 卡片顶部显示正确选项
  - 引导问题：「请选择最接近你现在理解的选项：」
  - 提供 3-4 个预设模板
  - 最后一个选项「我还是不太理解为什么这是对的：」支持自由输入

**Step 5 - 自我诊断：**
- 引导问题：「回顾你的解题过程，你认为主要错在：」
- 交互方式：单选 + 条件输入
- 提供 4-5 个错误类型选项：定位句找错了、答案句理解不对、选项主干内容不明确、选项细节信息理解错误、其他
- 选择「其他」时展开输入框

**Step 6 - 补充说明：**
- 引导问题：「还有其他想补充的吗？（选填）」
- 交互方式：纯文本输入，非必填
- 提供输入提示：「例如：有哪些词汇/句子影响了你的理解？你在解题时有什么特殊的思考？」

### 3.3 DiagnosisPage（诊断结果页）

#### 布局结构
采用**左右分栏布局**，比例为 45:55。

**左侧区域 - 复盘步骤详情：**

垂直排列 6 个步骤的结果卡片，每个卡片包含：
- 步骤名称和正误状态图标（✓ 或 ✗）
- 正确的步骤：卡片保持收起状态，仅显示步骤名称和绿色 ✓ 图标
- 错误的步骤：卡片自动展开，显示详细信息：
  - 「你选择了：」+ 用户的选择内容
  - 「问题所在：」+ 具体问题描述
  - 「应该如何思考：」+ 正确的思维方式指导

**右侧区域 - AI 分析与能力提升：**

分为上下两个模块：

**上方模块 - AI 问题分析：**
- 模块标题：「AI 问题分析」
- 错误类型标签：显示诊断出的错误类型（如「答案句理解偏差」）和层级（如「Level 3」）
- 分析正文：AI 生成的详细错因分析，解释为什么会犯这个错误，原文和选项之间的关系是什么

**下方模块 - 能力提升建议：**
- 模块标题：「能力提升建议」
- 缺失能力标签：如「📚 缺失能力：因果关系识别」
- 提升方法列表：编号列表形式，提供 3-5 条具体可操作的练习建议

**底部导航区：**
- 左侧「返回首页」按钮（灰色次按钮）
- 右侧「下一题」按钮（蓝色主按钮）

## 四、视觉规范

### 4.1 配色方案

| 用途 | 颜色名称 | 色值 | 使用场景 |
|------|---------|------|---------|
| 正确/成功 | 绿色 | #22C55E | 正确选项背景、✓ 图标 |
| 错误/失败 | 红色 | #EF4444 | 错误选项背景、✗ 图标 |
| 主要操作 | 蓝色 | #3B82F6 | 主按钮、进度条已完成节点、链接 |
| 次要操作 | 灰色 | #6B7280 | 次按钮、禁用状态 |
| 页面背景 | 白色 | #FFFFFF | 页面主背景 |
| 卡片背景 | 浅灰 | #F9FAFB | 文章区域、卡片背景 |
| 主文字 | 深灰 | #1F2937 | 正文、标题 |
| 次文字 | 中灰 | #6B7280 | 说明文字、提示文字 |
| 边框 | 浅灰 | #E5E7EB | 卡片边框、分割线 |

### 4.2 组件规范

**按钮：**
- 主按钮：蓝色背景、白色文字、圆角 8px、内边距 12px 24px
- 次按钮：白色背景、灰色边框、灰色文字、圆角 8px
- 悬停效果：颜色加深 10%
- 禁用状态：透明度 50%，不可点击

**卡片：**
- 白色背景、圆角 12px、阴影 `0 2px 8px rgba(0,0,0,0.1)`
- 内边距 20px
- 复盘步骤卡片：最大宽度 600px，水平居中

**选项：**
- 默认：白色背景、灰色边框（1px）、圆角 8px
- 悬停：浅蓝色背景
- 选中：蓝色边框（2px）
- 正确：绿色背景、绿色边框
- 错误：红色背景、红色边框

### 4.3 响应式设计

| 屏幕宽度 | 布局调整 |
|---------|---------|
| ≥1024px（桌面） | 左右分栏布局，比例按设计稿 |
| 768-1023px（平板） | 左右分栏布局，比例调整为 40:60 |
| <768px（手机） | 上下堆叠布局，文章区域默认折叠 |

## 五、交互动效规划

### 5.1 已确认的动效

| 交互场景 | 动效描述 |
|---------|---------|
| 页面切换 | 淡入淡出过渡，时长 300ms |
| 选项选中 | 边框颜色变化，时长 150ms |
| 答题结果 | 选项颜色渐变，时长 300ms |
| 折叠展开 | 高度动画 + 透明度变化，时长 250ms |
| 进度条 | 节点状态变化带缩放效果 |

### 5.2 预留的动效接口（未来迭代）

**复盘卡片划走动画：**
- 向右滑动：卡片向右滑出 + 淡出，表示确认当前步骤，自动进入下一步
- 向左滑动：卡片向左滑出 + 淡出，表示返回上一步
- 滑动过程中卡片跟随手指移动并轻微旋转
- 松手后根据滑动距离判断是否触发切换，未达到阈值则回弹

## 六、技术实现要点

### 6.1 前端技术栈
- 框架：React 18 + Vite
- 样式：Tailwind CSS
- 路由：React Router v6
- 状态管理：React Context（MVP 阶段足够）
- HTTP 请求：Fetch API 或 Axios
